# Документы для базы знаний (папка data/)

В эту папку кладутся **текстовые файлы .txt** в кодировке UTF-8. Все `.txt` из папки загружаются в векторную базу. По имени файла в коде задаётся поле `code` — по нему бот фильтрует поиск (например, только по ГПК РФ или по претензии).

## Текущие файлы и привязка к кодексу

| Файл | Код (фильтр в боте) |
|------|---------------------|
| **Исковое в суд по ГПК.txt** | ГПК РФ |
| **gpk_rf.txt** | ГПК РФ |
| **ИСКОВОЕ ЗАЯВЛЕНИЕ ПО АПК РФ.txt** | АПК РФ |
| **apk_rf.txt** | АПК РФ |
| **gk_rf1.txt**, **gk_rf2.txt**, **gk_rf3.txt** | ГК РФ |
| **ПРЕТЕНЗИЯ.txt** | претензия |

В вопросе можно писать «по ГПК», «по АПК», «ГК РФ», «претензия» — поиск ограничится нужным типом документов.

## Что класть в папку

- Регламенты, инструкции, шаблоны (письмо, претензия, иск).
- Выдержки из кодексов (ГПК, АПК, ГК РФ) — текстом в UTF-8.
- Формат: только **.txt**. PDF/DOCX предварительно конвертировать в текст.

После добавления или изменения файлов пересоберите векторную базу: удалите папку **`chroma_db/`** в корне проекта и снова запустите `python app.py`.

## Важно

Если добавляете новый файл с новым кодексом/типом, нужно прописать его в **`vector_store.py`** в словаре **`FILE_TO_CODE`** (имя файла без `.txt` → значение `code`), и при необходимости добавить ключевые слова в **`app.py`** в **`QUERY_TO_FILTER`**.
